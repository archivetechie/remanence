# RAO Vector Manifests

These JSON manifests pin the initial Section 13 values for the RAO 1.0
positive vectors. The original core vectors were generated by the Remanence
reference implementation. The additional Section 13.1 plaintext-suite fixtures
were generated by the independent Python verifier first, then checked by
`crates/remanence-format/tests/rao_vectors.rs`, so the expanded suite does not
freeze a value produced only by the Rust implementation.

The independent Python verifier rebuilds the positive plaintext streams without
using the Remanence Rust crates. The RAO-TV-D1 encrypted half is now a v2
recipient envelope pinned by the Rust deterministic test-vector entry point.

The same verifier carries the Section 14 plaintext-interop gate:

```sh
python3 tools/verify_rao_vectors_independent.py --check-plaintext-interop
```

That command requires GNU tar, bsdtar, and Python `tarfile`, and extracts every
positive plaintext vector byte-identically, including RAO-TV-P1, the plaintext
RAO-TV-D1, and the additional RAO 13.1 fixtures. In an environment where bsdtar
is not installed, `--allow-missing-bsdtar` runs the GNU tar and Python
`tarfile` portions while making the missing reader explicit.

The former v1 encrypted long-term-recovery drill was retired with RAO-TV-E1.
Its v2 publication replacement, including independent HPKE rederivation, is a
P2 deliverable; requesting the old verifier flag fails loudly in the meantime.

Negative-vector manifests with `status: complete` enumerate the Section 13.5
negative families. The Rust negative-vector tests assert both the exact case
IDs and the mapped errors, so dropping a required case fails the fixture gate.
